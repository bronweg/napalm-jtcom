TASK: Unify all port numbering across the entire project to 1-based numbering

GOAL

Perform a project-wide refactor so that port numbering is consistent everywhere.

After this change, the project must use:

- 1-based port numbering in all user-facing interfaces
- 1-based port numbering in the domain model
- 1-based port numbering in diffs, warnings, errors, logs, docs, examples, and tests

0-based numbering must no longer appear anywhere in the public or domain-facing API.

If 0-based numbering is still required internally for some very low-level vendor payload or parser edge case, it must be:
- strictly localized
- hidden
- converted immediately at the boundary
- never exposed outside that boundary

PRIMARY PRINCIPLE

There must be exactly one numbering convention across the project:

    PORTS ARE 1-BASED

This includes:
- Python model objects
- VlanConfig membership fields
- PortConfig.port_id
- driver internal canonical membership map
- diff output
- warnings
- exceptions
- Ansible input
- Ansible output
- tests
- README / docs / examples

MOTIVATION

The current mixed use of 0-based and 1-based port numbering is unacceptable and dangerous.

Even if technically correct in some places, it creates:
- constant mental overhead
- ambiguous examples
- avoidable bugs
- misleading diffs
- broken trust in the tool

This refactor must eliminate the ambiguity completely.

STRICT NON-GOALS

Do NOT:
- redesign Step 4 policies yet
- add new features
- change VLAN membership semantics
- change mode-change policy
- change delete semantics
- refactor unrelated logic for style only

This task is exclusively about unifying port numbering.

PROJECT-WIDE TARGET STATE

After this task:

1) PortConfig.port_id remains 1-based
2) VlanConfig membership port references become 1-based
3) All canonical per-port maps become 1-based
4) All diff structures use 1-based ports
5) All warnings/errors/logs use 1-based ports
6) All docs/examples/tests use 1-based ports
7) Any 0-based conversion, if still needed, is limited to vendor-transport edge only
