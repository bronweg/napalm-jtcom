TASK: Remove duplicated "Port N" -> port ID parsing logic and centralize it in one shared helper

GOAL

Eliminate duplicated port-name parsing logic across the project by introducing a single shared helper for converting JTCom-style port names like:

- "Port 1"
- "Port 5"

into canonical 1-based port IDs.

This task is a cleanup/refactor for consistency and future safety.
It must not change runtime behavior.

CONTEXT

After Step 5B, there are now at least two places that perform very similar parsing/conversion work:

- _port_config_id(...)
- _port_name_to_id(...)

Even if both currently work, this duplication is dangerous because future changes may update one path and not the other.

The project now has a strict rule:
- ports are 1-based everywhere in the canonical/domain model

Therefore the conversion from vendor naming ("Port N") to canonical port ID must have exactly one authoritative implementation.

STRICT NON-GOALS

Do NOT:
- redesign the canonical model
- redesign readback verification
- redesign driver flow
- change Ansible API
- change Step 4 or Step 5 semantics
- broaden this into a generic parser refactor
- change behavior of unrelated helpers

This task is only about:
- de-duplicating port name → port ID conversion
- making the code clearer and safer

FILES TO MODIFY

Primary:
- src/napalm_jtcom/utils/vlan_membership.py
- src/napalm_jtcom/driver.py

Possibly:
- any other file that parses "Port N" strings directly
- tests that currently target the old helper names/locations

Tests:
- tests/unit/test_vlan_membership_apply.py
- add/update any targeted test file if needed

DESIRED END STATE

After this task:

1) There is exactly one shared helper that converts vendor-style port name to canonical 1-based port ID
2) All code paths that need this conversion use that helper
3) Helper naming is clear and consistent
4) No duplicate "Port N" parsing logic remains in tracked code
5) Runtime behavior is unchanged
6) Tests still pass

PART 2 — DEFINE ONE AUTHORITATIVE HELPER

Create or keep exactly one shared helper for this conversion.

Recommended location:
- src/napalm_jtcom/utils/vlan_membership.py

Reason:
- Step 5A/5B already established vlan_membership.py as a central semantic layer around canonical port/VLAN state

Recommended final name:
- port_name_to_id(...)

If you want to keep it private:
- _port_name_to_id(...)

The important thing is:
- use "id", not "index"
- name must clearly imply 1-based canonical port ID
- do not keep two differently named helpers that do the same thing

Required behavior:
Input:
- "Port 1" -> 1
- "Port 5" -> 5

Invalid input:
- malformed port names should raise clear ValueError
- do not silently return nonsense

If current implementation already does this correctly, keep the behavior and just centralize usage.
