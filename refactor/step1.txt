PHASE 1 — STAGE 1
Introduce canonical VLAN membership operations (add/remove/set) in the data model, preserve backward compatibility,
and do not change runtime behavior yet.

OBJECTIVE

Refactor the VLAN configuration model so that VLAN membership is no longer represented only as an implicit "replace"
via tagged_ports / untagged_ports lists.

Introduce an explicit canonical operations model:

- tagged_add
- tagged_remove
- tagged_set

- untagged_add
- untagged_remove
- untagged_set

This stage is model-only.
Do NOT modify driver behavior, device calls, action plugin semantics, or Ansible UX in this stage.

WHY THIS STAGE EXISTS

Current model only supports:
- tagged_ports
- untagged_ports

This implicitly behaves as a full replacement model and is unsafe for trunk permit lists.

We want the model layer to support:
- additive patching
- subtractive patching
- explicit replacement
- future conflict detection
- future access↔trunk mode-change policy

But we do not want to change actual switch behavior until the next stage.

STRICT NON-GOALS FOR THIS STAGE

Do NOT:
- change set_vlans()
- change apply_device_config()
- change get_vlans()
- change any device CGI calls
- change action plugin behavior
- change Ansible examples
- change current switch-side behavior

This stage must be a safe preparatory refactor only.

REQUIRED MODEL CHANGES

1) Update VlanConfig dataclass

Current VlanConfig contains:
- vlan_id
- name
- tagged_ports
- untagged_ports
- state

Keep all existing fields.

Add these new optional fields:

- tagged_add: list[int] | None = None
- tagged_remove: list[int] | None = None
- tagged_set: list[int] | None = None

- untagged_add: list[int] | None = None
- untagged_remove: list[int] | None = None
- untagged_set: list[int] | None = None

Keep:
- tagged_ports
- untagged_ports

They must remain fully supported for backward compatibility.

2) Introduce normalized_membership() method on VlanConfig

Add a method to VlanConfig:

def normalized_membership(self) -> dict[str, dict[str, set[int] | None]]:
    ...

Return structure MUST be exactly this logical shape:

{
  "tagged": {
    "add": set[int],
    "remove": set[int],
    "set": set[int] | None,
  },
  "untagged": {
    "add": set[int],
    "remove": set[int],
    "set": set[int] | None,
  },
}

SEMANTIC RULES FOR NORMALIZATION

A) Tagged side
- If tagged_set is provided:
    - normalized["tagged"]["set"] = set(tagged_set)
    - normalized["tagged"]["add"] = empty set
    - normalized["tagged"]["remove"] = empty set
- Else if legacy tagged_ports is provided or non-empty:
    - treat legacy tagged_ports as tagged_set
- Else:
    - normalized["tagged"]["set"] = None
    - normalized["tagged"]["add"] = set(tagged_add or [])
    - normalized["tagged"]["remove"] = set(tagged_remove or [])

B) Untagged side
- If untagged_set is provided:
    - normalized["untagged"]["set"] = set(untagged_set)
    - normalized["untagged"]["add"] = empty set
    - normalized["untagged"]["remove"] = empty set
- Else if legacy untagged_ports is provided or non-empty:
    - treat legacy untagged_ports as untagged_set
- Else:
    - normalized["untagged"]["set"] = None
    - normalized["untagged"]["add"] = set(untagged_add or [])
    - normalized["untagged"]["remove"] = set(untagged_remove or [])

C) Missing values
- Missing operation fields must normalize to empty sets, not None
- Only the explicit replacement operation uses None to mean “no replacement requested”

VALIDATION RULES

Implement validation in VlanConfig.__post_init__ (or a helper called from it).

Validation must fail fast with ValueError on invalid combinations.

Tagged validation:
- tagged_set cannot coexist with tagged_add
- tagged_set cannot coexist with tagged_remove
- legacy tagged_ports cannot coexist with tagged_add
- legacy tagged_ports cannot coexist with tagged_remove
- legacy tagged_ports cannot coexist with tagged_set

Untagged validation:
- untagged_set cannot coexist with untagged_add
- untagged_set cannot coexist with untagged_remove
- legacy untagged_ports cannot coexist with untagged_add
- legacy untagged_ports cannot coexist with untagged_remove
- legacy untagged_ports cannot coexist with untagged_set

PORT VALUE VALIDATION

Validate every port identifier that appears in:
- tagged_ports
- untagged_ports
- tagged_add
- tagged_remove
- tagged_set
- untagged_add
- untagged_remove
- untagged_set

Rules:
- must be int
- must be >= 0
- duplicates are allowed in input but must collapse during normalization
- do not validate upper bound here (driver/device-specific)

VLAN VALIDATION

Preserve existing VLAN validation:
- vlan_id must remain 1..4094

STATE VALIDATION

Preserve existing state validation semantics:
- state remains "present" | "absent"

BACKWARD COMPATIBILITY REQUIREMENTS

Legacy playbooks must still work unchanged.

That means:
- tagged_ports still accepted
- untagged_ports still accepted
- old callers constructing VlanConfig(tagged_ports=[...]) still work
- normalized_membership() must interpret legacy fields as replacement semantics

IMPORTANT BACKWARD-COMPATIBILITY DETAIL

Legacy fields mean REPLACE, not MERGE.

So:
- tagged_ports -> tagged_set
- untagged_ports -> untagged_set

Do not reinterpret legacy fields as add/remove.

NO RUNTIME BEHAVIOR CHANGE

This stage must not alter current behavior anywhere else in the codebase.

The only purpose is:
- extend the model
- add normalization API
- add validation
- preserve backward compatibility
